
Последовательность команд которая, работает для git:

git add . - индексирует файлы, добавляет файлы под контроль git

git remote add blog https://github.com/kirillvoyshnarovich/blog-on-django.git - добавляет удаленный репозиторий, под сокращенным именем "blog", потом обращаться можно к этому сокращенному имени вместо полного URL

git commit -m "my blog on django, first commit" - делаем коммит (снимок), состояния файлов которое было на момент, когда выполнялась команда - "git add"

git status - смотрим файлы которые находятся под версионным контролем, какие  нет

git push blog master - заливаем наш проект(или наши изменения) на удаленный сервер
 по адресу - blog, в ветку master.
